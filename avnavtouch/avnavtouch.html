avnavTouch

avnavTouch  basiert auf dem wunderbaren avnav von Andreas Vogel und erweitert die Funktionalität um folgende Punkte:

avNavTouch Funktionsübersicht

  • Aktualisierte OS Version: Rapsbian Stretch (Jessie bekommt leider keine Updates mehr)

  • Erstinstallation via PINN startet Touchmodus, ist aber jederzeit umschaltbar zum Servermodus (Skript aufrufen, Modus umstellen booten, fertig)

  • Touchmodus: startet ohne Systemmeldungen, sondern mit nettem avnav Screenshot und geht direkt in die Oberfläche

  • Touchmodus: On-Screen Tastatur OnBoard, keine Hardware nötig (außer, man hat zu dicke Finger)

  • Touchmodus: automatischer Kartenabgleich bei Einstecken eines USB Sticks

  • Touchmodus: Daemon "twofing" sorgt für eine bequeme Mehr-Finger-Bedienung, also Scrollen mit zwei Fingern durch Wegpunkte oder AIS Listen, Pinch-To-Zoom in der Karte, etc. geht alles

  • Touchmodus: Original avnav WebUI im Fullscreen Mode, es wird zusätzlich auf einem 2. Desktop die das Script avnav_setup.sh gestartet, so dass man trotzdem auch mal eben was umstellen oder auf eine Konsole wechseln kann (via VNC oder alternativ mit Tastatur)
    -> Wechsel auf den 2.Desktop mit STR-ALT-PfeilNachRechts

  • Touchmodus: VNC Server aktivierbar, um via WLAN auf die Oberfläche zugreifen zu können

  • Servermodus: normale Konsolenmeldungen, wie gewohnt inkl. Login (User: pi/ Pass: raspberry, wie gewohnt)

  • in beiden Modi: bei Einstecken eines RTL-SDR Receivers (aktuell konfiguriert für NooElec NESDR SMArt, aber jederzeit via udev erweiterbar, siehe unten) wird automatisch rtl_ais gestartet und stellt AIS Daten bereit

  • für RTL-SDR: das Tool kalibrate-rtl steht zur Einmessung des Frequenzoffsets bereit

  • Zentrales Konfigurationsscript: sudo avnav_setup.sh
    Dieses Tool steht im Touchmodus auf dem zweiten Desktop (erreichbar über STRG-ALT-PfeilNachRechts) dauerhaft zur Verfügung. Alternativ kann es auch im Servermodus mit dem genannten Befehl gestartet werden.
